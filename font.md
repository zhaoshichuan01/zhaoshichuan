# Claude 生态与前端基础全景指南

本文档旨在为 Java 开发者提供一份关于 Anthropic 最新技术（Claude Sonnet 4.5, Claude Code）及前端核心运行环境的完整参考手册。

---

## 第一部分：Claude 模型家族 (Model Family)

### 1. Claude Sonnet 4.5 (最新旗舰)
**发布时间**：2025 年 9 月 29 日
**核心定位**：目前世界上最强的编程模型、Agent 构建模型及计算机操作模型。

#### 核心能力突破
*   **编程能力 (Coding)**：
    *   **SOTA 表现**：在 SWE-bench Verified（真实软件工程任务）中刷新纪录。
    *   **0% 编辑错误率**：在内部代码编辑基准测试中，错误率从上一代（Sonnet 4）的 9% 降至 0%，极大提升了修改代码的安全性。
    *   **超长续航**：能够保持专注处理超过 **30 小时** 的复杂、多步骤编程任务。
*   **计算机使用 (Computer Use)**：
    *   **OSWorld 基准**：得分 **61.4%**（前代 Sonnet 4 为 42.2%）。
    *   **能力描述**：具备极强的操作浏览器、填写表格、使用桌面软件的能力，能像人类一样操作 GUI。
*   **领域推理**：在金融、法律、医学和 STEM 领域表现超越 Opus 4.1。

### 2. Claude Haiku (极速轻量版)
**定位**：速度最快、成本最低、最轻量级的版本。
**适用场景**：
*   需要毫秒级响应的实时交互。
*   处理海量数据的文本提取、翻译。
*   成本敏感型任务。
    **现状**：虽然是轻量版，但最新一代 Haiku 在许多指标上已超越旧版旗舰（如 Claude 3 Opus）。

## 第二部分：Claude Code (AI 编程 Agent)

### 1. 定义与原理
**Claude Code** 是 Anthropic 官方推出的、运行在终端（Terminal）中的 AI 编程代理工具。

*   **技术栈**：基于 **Node.js** 开发的命令行工具 (CLI)。
*   **驱动模型**：默认由 **Claude Sonnet 4.5** 驱动。
*   **运行原理**：
    *   它是一个安装在你本地电脑上的程序（类似 Maven 或 Git）。
    *   它拥有**文件系统读写权限**和**Shell 执行权限**。
    *   它充当云端 AI 大脑的“手脚”，直接修改本地文件、运行测试命令。

### 2. 核心功能与新特性 (2025.09 更新)
*   **自主任务执行**：你下达复杂指令（如“重构这个模块”），它自主规划 -> 搜索文件 -> 修改代码 -> 运行测试 -> 修复错误。
*   **Checkpoints (检查点)**：支持存档与回滚。如果 AI 改坏了代码，可以瞬间恢复到上一个状态。
*   **VS Code Extension**：除了终端版，现在也有了官方 VS Code 插件。
*   **Claude Agent SDK**：官方开放了底层基础设施，开发者可以用此 SDK 构建自己的 Agent。

### 3. 使用指南
1.  **安装**：`npm install -g @anthropic-ai/claude-code`
2.  **启动**：在项目根目录下运行 `claude`
3.  **认证**：首次运行需浏览器授权。
4.  **交互**：直接输入自然语言指令，如 `run tests and fix the bugs in UserServiceImpl`。

### 4. Claude Code vs. Cursor / Trae

| 维度 | Claude Code | Cursor / Trae (字节) |
| :--- | :--- | :--- |
| **产品形态** | **CLI 工具 (命令行)** | **IDE (集成开发环境)** |
| **核心定位** | **Agent (自主代理)** | **Copilot (辅助驾驶)** |
| **工作方式** | "外包模式"：你给任务，它后台跑，不需要你盯着。 | "结对模式"：你写代码，它实时补全、回答问题。 |
| **优势** | 擅长长链路、繁琐任务（重构、升级依赖、修测试）。 | 擅长即时交互，代码编写手感好，可视化强。 |
| **模型支持** | 强制绑定 Claude 模型。 | Cursor 支持多模型；Trae 主打豆包模型。 |

## 第三部分：前端基础环境 (Java 开发者视角)

为了理解 Claude Code 的运行基础，我们将 Node.js 生态核心概念与 Java 后端生态进行对标。

### 1. 核心概念对照表

| 概念 | 全称 | Java 生态对标 | 核心作用详解 |
| :--- | :--- | :--- | :--- |
| **Node.js** | - | **JDK / JVM** | **运行环境**。<br>让 JavaScript 脱离浏览器，在服务器/电脑上运行。没有它，JS 只能写网页交互；有了它，JS 可以读写文件、做后端服务、运行 CLI 工具（如 Claude Code）。 |
| **npm** | Node Package Manager | **Maven / Gradle** | **包管理与构建工具**。<br>1. **依赖管理**：`package.json` ≈ `pom.xml`。<br>2. **仓库下载**：下载包到 `node_modules` ≈ Maven 本地仓库 (`.m2`)。<br>3. **脚本执行**：`npm run build` ≈ `mvn package`。 |
| **NVM** | Node Version Manager | **SDKMAN! / jenv** | **版本管理工具**。<br>允许本机同时安装多个 Node 版本（如 v18, v23），并快速切换。解决不同项目（老项目用 v14，新项目用 v20）的兼容性问题。 |

### 2. 协作流程示例
当你接手一个前端项目（如 `trade.tool-fe`）或使用 JS 工具时：

1.  **环境准备 (NVM)**：
    *   命令：`nvm use 23`
    *   作用：切换到正确的“JDK”版本（你当前本地是 v23.11.0）。
2.  **依赖安装 (npm)**：
    *   命令：`npm install`
    *   作用：读取 `package.json`，下载所有依赖包（相当于 `mvn install`）。
3.  **运行工具 (Node.js)**：
    *   命令：`npm run dev` (启动项目) 或 `claude` (启动工具)
    *   作用：Node.js 进程启动，执行 JS 代码。
